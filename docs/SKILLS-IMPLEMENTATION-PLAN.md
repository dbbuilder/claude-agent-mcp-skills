**Detection Patterns:**

1. **SQL Injection (TypeScript)**
```typescript
// BAD - String concatenation
const query = `SELECT * FROM users WHERE id = ${userId}`;

// BAD - Template literals with user input
db.query(`SELECT * FROM users WHERE email = '${email}'`);

// GOOD - Parameterized queries
db.query('SELECT * FROM users WHERE id = ?', [userId]);
```

2. **XSS (React/Vue)**
```typescript
// BAD - dangerouslySetInnerHTML
<div dangerouslySetInnerHTML={{__html: userInput}} />

// BAD - v-html in Vue
<div v-html="userInput"></div>

// GOOD - Escaped by default
<div>{userInput}</div>
```

3. **Hardcoded Secrets**
```typescript
// BAD - Patterns to detect
const API_KEY = "sk-1234567890abcdef";
const PASSWORD = "password123";
const JWT_SECRET = "my-secret-key";
process.env.AWS_SECRET = "AKIA..."; // Should be from env

// Detection regex
/(?:api[_-]?key|password|secret|token)\s*[=:]\s*["'][^"']+["']/gi
```

4. **Insecure Crypto**
```csharp
// BAD - MD5/SHA1
var hash = MD5.Create().ComputeHash(data);

// BAD - Weak random
Random rand = new Random();
var token = rand.Next();

// GOOD - Strong crypto
using var rng = RandomNumberGenerator.Create();
var hash = SHA256.Create().ComputeHash(data);
```
